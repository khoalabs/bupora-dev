🧱 DATABASE chuẩn production-lite
🟢 1. users (internal only)
users (
    id SERIAL PRIMARY KEY,
    username TEXT UNIQUE NOT NULL,
    display_name TEXT NOT NULL,
    role TEXT NOT NULL, -- admin | editor
    created_at TIMESTAMP DEFAULT now()
)

🟢 2. categories
categories (
    id SERIAL PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    slug TEXT UNIQUE NOT NULL
)

🟢 3. posts (CORE TABLE)
posts (
    id SERIAL PRIMARY KEY,
    title TEXT NOT NULL,
    slug TEXT UNIQUE NOT NULL,
    content TEXT NOT NULL,
    category_id INT REFERENCES categories(id),
    author_id INT REFERENCES users(id),
    status TEXT DEFAULT 'published', -- draft | review | published
    read_time INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT now(),
    updated_at TIMESTAMP DEFAULT now()
)

🟢 4. tags (optional nhưng rất nên có)
tags (
    id SERIAL PRIMARY KEY,
    name TEXT UNIQUE NOT NULL,
    slug TEXT UNIQUE NOT NULL
)

🟢 5. post_tags (many-to-many)
post_tags (
    post_id INT REFERENCES posts(id),
    tag_id INT REFERENCES tags(id),
    PRIMARY KEY (post_id, tag_id)
)


⚡ CACHE DESIGN (cực quan trọng với VPS $5)

🟢 1. Post cache > 👉 TTL: 1h – 24h
post:{slug}
post:{id}

🟢 2. Category list cache
category:breeds
category:care
category:lifestyle

🟢 3. Homepage cache > 👉 TTL ngắn hơn (30s – 5m)
homepage:latest

🧠 READ FLOW (quan trọng nhất)
Request → Cloudflare → Nginx → Redis → Postgres (fallback)

🟢 WRITE FLOW (admin/editor)
Admin API →
    Postgres update →
    invalidate Redis keys →
    (optional) Cloudflare purge
